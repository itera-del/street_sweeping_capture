# video_cliper：Tkinter 视频裁剪工具 — 设计说明

**日期：** 2026-05-03  
**范围：** `E:\Projects\video_cliper` 桌面应用（单段、stream copy、静音预览）

---

## 1. 目标与成功标准

### 1.1 目标

提供基于 **tkinter** 的 Windows 桌面小工具：用户打开本地视频，通过**静音画面预览**与**进度条/播放/逐帧**定位，使用按钮设定**入点/出点**（不设手填时间编辑框），将 `[start, end)` 区间以 **FFmpeg stream copy（不重编码）** 导出为新文件；导出保留**第一路视频 + 第一路音频（若存在）**；输出封装与扩展名**默认与源文件一致**；记住**上次导出目录**；另存为对话框允许用户修改文件名。

### 1.2 成功标准

- PATH 中可调用 `ffmpeg` 与 `ffprobe` 时，常见 MP4/MOV 与无音轨样例可完成「打开 → 打点 → 导出 → 播放验证时长」。
- 无 FFmpeg 时给出明确错误提示，不崩溃。
- 导出失败时用户可看到 `ffmpeg` stderr 摘要以便排查。
- UI 在导出过程中保持可响应（后台线程执行子进程）。

---

## 2. 约束与非目标

### 2.1 硬性约束

- **不重编码**：导出固定 `-c copy`；失败**不**自动降级为重编码。
- **FFmpeg 发现策略**：仅使用系统 **PATH**；**不**提供自定义 `ffmpeg.exe` 路径配置。
- **预览**：**不**播放音频；仅处理画面帧用于选点。
- **时间输入**：**不**提供可编辑时间文本框；入/出点仅来自预览当前时间 + 按钮；可显示**只读**时间码。

### 2.2 当前版本非目标

- 批量多文件、多段一次导出、滤镜、内嵌 FFmpeg 安装包。
- 保留多路音频、字幕或其它数据轨（仅 `0:v:0` 与可选 `0:a:0`）。
- 手填/微调时间文本、自定义 ffmpeg 路径。

| 组件 | 用途 |

## 4. 架构与模块

### 4.1 模块划分

1. **应用入口**  
   启动时检测 `ffmpeg` 与 `ffprobe` 可执行性；若不可用则弹出**非阻塞**警告说明原因。主窗口照常显示；在用户点击「打开视频」或「导出片段」时**再次检测**，仍不可用则阻断该次操作并提示，避免在无工具链下进入无效流程。

2. **`ffmpeg_tools`（名称可调整）**  
   - `check_binaries()`：验证 PATH。  
   - `probe_media(path)`：封装 `ffprobe -print_format json`，解析 `format.duration`、视频流 `fps`、是否存在 `audio` 流。  
   - `build_export_command(src, dst, start, end)` / `run_export(...)`：构造并执行 `ffmpeg` 参数（见第 5 节）。

3. **`preview`（或等价类）**  
   封装 `cv2.VideoCapture`：`open`、`seek_seconds(t)`、`read_frame_bgr`、`step_frame(delta_frames)`、`current_time_sec`；与 UI 解耦，便于测试与维护。

4. **主窗口 UI**  
   组合工具栏、Canvas 预览、进度条（Scale）、只读时间显示、状态栏；绑定预览控制器与导出动作。

5. **配置**  
   持久化 `last_export_dir`；路径见第 7 节。

### 4.2 预览缩放

按主窗口预览区宽度**等比缩放**显示，保持宽高比；过大分辨率不得撑爆布局。

## 5. FFmpeg 与数据流

### 5.1 剪切语义

区间 **`[start, end)`**：起点包含，终点不包含；长度 `end - start` 秒。

### 5.2 命令策略（固定）

为优先满足「快、可对齐关键帧」：

```text
ffmpeg -hide_banner -y -ss <start> -i <src> -t <end-start> \
  -map 0:v:0 [若存在音频: -map 0:a:0] \
  -c copy <dst>
```

- **`-ss` 在 `-i` 之前**；**`-t`** 为 `end - start`。  
- **音频**：若探测到至少一路 audio，则 `-map 0:a:0`；否则不 map 音频。  
- **字幕与其它轨**：默认不 map。

### 5.3 数据流

1. 打开：`ffprobe` → 成功则 `VideoCapture.open`，初始化 `duration`、fps 回退链、是否有音频。  
2. 预览：UI 时间 `t` → `seek` → 读帧 → RGB → Canvas。OpenCV seek 精度不保证与 FFmpeg 完全一致，以可操作为准；**导出以 UI 记录的秒数为准**。  
3. 导出：后台线程执行 `ffmpeg`；主线程通过 `after` 更新状态与错误展示。

### 5.4 另存为与命名

- **每次导出**弹出另存为。  
- 默认目录：`last_export_dir`；若未配置，默认使用**源文件所在目录**。  
- 默认文件名：`<源主干>_clip` + **与源相同扩展名**（后缀字符串实现时固定为 `_clip` 或等价常量）。用户可改文件名；若用户修改扩展名，**仍按用户路径保存**，程序不强制改回；copy/remux 风险由用户承担。

## 6. 界面与交互（摘要）

自上而下：

1. **工具栏**：打开视频；设入点/出点；跳到入点/出点；上一帧/下一帧；播放/暂停；导出（入点 < 出点时启用）。  
2. **预览区**：Canvas 显示当前帧；未打开时占位文案。  
3. **时间轴**：水平 Scale，`0 … duration`；拖动即 seek 并刷新帧；拖动时暂停播放。  
4. **只读时间行**：当前时间、总长、入点、出点。  
5. **状态栏**：就绪 / 导出中 / 错误摘要。

**播放**：`after` 循环，步长 `1/fps` 秒；fps 优先 `ffprobe` 视频流，其次 `CAP_PROP_FPS`，再默认 `25`。播放到 `duration` 暂停在末帧附近。

**逐帧**：播放中建议先自动暂停；按 fps 反算秒步进，clamp 到 `[0, duration]`。

**出点 ≤ 入点**：导出禁用，状态栏提示。

## 7. 配置持久化

- **建议路径**：`%USERPROFILE%\.video_cliper\config.json`。  
- **字段**：`last_export_dir`（字符串）。  
- 首次运行无文件则视为无配置。

## 8. 错误处理

| 场景 | 行为 |
|------|------|
| PATH 无 `ffmpeg`/`ffprobe` | 明确提示安装或激活 conda 环境（如 `video_process`） |
| `ffprobe` 失败 | 错误对话框 + 可选展示 stderr |
| OpenCV 无法打开 | 提示无法预览该文件 |
| `ffmpeg` 非零退出 | 对话框展示 stderr 尾部（可滚动） |
| 导出目录不可写 | 提示失败原因 |

---

## 9. 测试建议

- **手工**：H.264+AAC MP4、常见 MOV、无音轨视频；异常路径含无 PATH、损坏文件、出点≤入点、只读目录。  
- **单元测试（可选）**：`ffprobe` JSON 解析、导出 argv 构造快照；不对 tkinter 线程做重测。
